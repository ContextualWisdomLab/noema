# 파일럿 온보딩 진행 기록

## 기록 규칙
- `완료` 처리된 행만 최종 `유료 파일럿` 카운트에 반영한다.
- 각 항목은 `운영 전환 승인일`, `분석 데이터 경로`, `지원 채널 합의`, `증빙 출처: production`, `계약/매출 증빙 경로`가 있어야 `완료` 처리된다.
- `example`, `localhost`, `.local` 값은 샘플로 간주하며 완료 파일럿으로 인정하지 않는다.
- `검증 증빙`은 해당 항목 하위에 파일 경로 또는 로그 추적키를 남긴다.

## 항목 1
- 고객명:
- 시작일:
- 담당자(공급자/고객):
- 환경:
- NOEMA URL:
- 계약/제안 단계:
- [ ] 가격 합의
- [ ] SLA/지원 범위 합의
- [ ] 이용약관 확정
- 지원 채널 합의:
- 증빙 출처:
- 계약/매출 증빙 경로:

## 기술 검증
- [ ] `/health` 확인 (`ok: true`, `trace_id`)
- [ ] `/exchange` 정상 토큰 교환 1건 이상
- [ ] 실패 응답이 표준 에러 코드로만 발생
- [ ] `x-trace-id`, `x-latency-ms` 헤더 확인
- [ ] 장애 대응 알림 규칙 적용

## KPI 증빙
- 분석 데이터 경로:
- `exchange_failure_rate`:
- `exchange_p95_latency_ms`:
- 30일 구간 충족 여부:
- [ ] 실패율 <= 0.02
- [ ] p95 < 300

- 운영 전환 승인일:
- 온보딩 완료일:
- [ ] 7일/14일 회고 완료

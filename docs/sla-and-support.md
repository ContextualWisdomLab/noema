# Noema SLA 및 지원 정책(초안)

## 제공 범위
- `/health`: API 가용성 모니터링
- `/exchange`: OIDC → GitHub App 설치 토큰 교환

## 운영 목표(초안)
- SLA: 99.0% (월간, 계획 가동시간 기준)
- 핵심 KPI
  - `/exchange` 실패율 30일 누적 2% 이하 목표
  - `/exchange` p95 응답 300ms 미만 목표

## 지원 채널
- 우선순위 이슈 접수: Slack/이메일
- 긴급성 분류
  - P0: 인증/토큰 발급 전체 장애
  - P1: 단일 조직 장애 또는 반복적 인증 실패
  - P2: 기능 저하/문서/운영 이슈

## 장애 대응 시간
- P0: 15분 내 초기 응답, 2시간 내 임시 완화 조치
- P1: 1시간 내 임시 대응, 24시간 내 영구 조치 계획
- P2: 1영업일 내 업데이트, 다음 주기에 반영

## 제외 항목
- GitHub API 자체 장애
- 고객이 직접 관리하는 OIDC 토큰 발급 파이프라인의 스크립트 오류
- 네트워크 단절(고객 측)
